# v-011.04 — Rusticle Config & Bridge Commands

Parent: [v-011-kairn-spec.md](v-011-kairn-spec.md)

## Overview

kairn's configuration is a rusticle script, not JSON. The user writes
`.kairnrc.tcl` files using the same language used for keybindings,
themes, and automation. Bridge commands expose kairn internals (buffers,
editor, windows, terminal, themes, Kiro AI) to the scripting layer.

This spec covers:
- Config file discovery and loading order
- Embedded default config compiled into the binary
- `kairn --init-config` template generation
- Bridge command registry and individual command APIs
- Command manifest for static validation
- Keybinding system via the `bind` command
- Theme loading via rusticle
- Concrete Rust types

## Config file loading

### Search order

Config files are evaluated in order. Later files override earlier ones.

1. **Embedded defaults** — `include_str!("defaults.tcl")` compiled into
   the binary. Always runs first. Defines sane defaults for all settings.
2. **Global config** — `~/.kairnrc.tcl`. User's personal preferences.
3. **Project config** — `$PWD/.kairnrc.tcl`. Project-specific overrides.

Each file is a complete rusticle script. They share the same interpreter
instance, so project config can reference or override values set by the
global config.

### Loading sequence

```
┌─────────────────────┐
│  Interpreter::new()  │
└──────────┬──────────┘
           │
           ▼
┌─────────────────────┐
│  load_manifest()    │  ← register bridge command signatures
└──────────┬──────────┘
           │
           ▼
┌─────────────────────┐
│  eval(DEFAULTS_TCL) │  ← include_str!("defaults.tcl")
└──────────┬──────────┘
           │
           ▼
┌─────────────────────┐
│  eval(~/.kairnrc.tcl)│  ← if exists
└──────────┬──────────┘
           │
           ▼
┌─────────────────────┐
│  eval($PWD/.kairnrc) │  ← if exists
└──────────┬──────────┘
           │
           ▼
┌─────────────────────┐
│  validate + extract │  ← read context variables into Config
└─────────────────────┘
```

### Error handling

- If `~/.kairnrc.tcl` has a syntax error, kairn prints the error with
  source location and exits with code 78 (EX_CONFIG).
- If `$PWD/.kairnrc.tcl` has a syntax error, same behavior.
- Validation errors (type mismatches on declared variables) are fatal.
- The embedded defaults are compile-time verified and cannot fail.

### Explicit override

`kairn --config path/to/file.tcl` loads an explicit config after the
project config. This is useful for testing alternate configurations.

## Embedded default config

The file `src/config/defaults.tcl` is compiled into the binary via
`include_str!`. It defines the `kairn` context with all settings and
their type declarations, default keybindings, and the default theme.

```tcl
# src/config/defaults.tcl — compiled into binary

context kairn {
    declare keymap     : enum {vi emacs classic}
    declare tab-width  : int
    declare line-numbers : bool
    declare auto-save  : bool
    declare theme      : string
    declare shell      : string
    declare kiro-command : string

    set keymap       vi
    set tab-width    4
    set line-numbers true
    set auto-save    false
    set theme        "gruvbox-dark"
    set shell        ""
    set kiro-command "kiro-cli"
}

# Default keybindings
bind "ctrl+q"           { editor quit }
bind "ctrl+s"           { buffer save }
bind "ctrl+b"           { window toggle-tree }
bind "ctrl+tab"         { window cycle-focus }
bind "ctrl+p"           { editor open-search }
bind "ctrl+d"           { editor diff }
bind "ctrl+g"           { editor git-log }
bind "ctrl+shift+down"  { editor cycle-mode next }
bind "ctrl+shift+up"    { editor cycle-mode prev }
bind "f1"               { editor help }
bind "f3"               { window focus tree }
bind "f4"               { window focus editor }
bind "f5"               { window focus terminal }
bind "f6"               { window toggle-left }
bind "f11"              { window refresh-tree }
bind "f12"              { window redraw }
bind "alt+left"         { terminal prev-tab }
bind "alt+right"        { terminal next-tab }
bind "ctrl+x n"         { terminal new-kiro }
bind "ctrl+x t"         { terminal new-shell }
bind "ctrl+x k"         { terminal close-tab }
bind "ctrl+x ctrl+s"    { buffer save }
bind "ctrl+x s"         { editor save-session }

# Default theme
theme load "gruvbox-dark"
```

A user's `~/.kairnrc.tcl` only needs to override what differs:

```tcl
# ~/.kairnrc.tcl — personal overrides
set kairn::keymap emacs
set kairn::tab-width 2
set kairn::theme "catppuccin"

bind "ctrl+s" { buffer save-all }
bind "ctrl+k ctrl+c" { editor comment-toggle }

theme load "catppuccin"
```

## `kairn --init-config`

Generates a template `~/.kairnrc.tcl` with all settings commented out
and annotated. Does not overwrite an existing file.

```tcl
# ~/.kairnrc.tcl — kairn configuration
# Generated by: kairn --init-config
#
# Uncomment and modify settings as needed.
# See: https://github.com/user/kairn/doc/config.md

# ── Editor settings ──────────────────────────────────────────────
# set kairn::keymap vi           ;# vi | emacs | classic
# set kairn::tab-width 4
# set kairn::line-numbers true
# set kairn::auto-save false
# set kairn::theme "gruvbox-dark"
# set kairn::shell ""            ;# empty = $SHELL
# set kairn::kiro-command "kiro-cli"

# ── Keybindings ──────────────────────────────────────────────────
# bind "ctrl+q"       { editor quit }
# bind "ctrl+s"       { buffer save }
# bind "ctrl+b"       { window toggle-tree }
# bind "ctrl+p"       { editor open-search }

# ── Theme ────────────────────────────────────────────────────────
# theme load "gruvbox-dark"
```

Implementation: the template is a second `include_str!("init_template.tcl")`
embedded in the binary. `--init-config` writes it to `~/.kairnrc.tcl`
if the file does not exist, or prints to stdout if it does.

## Bridge command registry

Bridge commands are kairn-specific commands registered in the rusticle
interpreter. They follow the same pattern as rusticle-tk: each command
is a closure capturing shared application state behind `Arc<Mutex<T>>`.

### Architecture

```
┌──────────────┐     register_fn()     ┌──────────────┐
│  BridgeRegistry │ ──────────────────→ │  Interpreter  │
│                │                      │               │
│  SharedState   │ ←── Arc<Mutex<>> ──→ │  closures     │
│  (App, Editor, │                      │  (buffer,     │
│   Buffers,     │                      │   editor,     │
│   Terminal)    │                      │   window...)  │
└──────────────┘                      └──────────────┘
```

All bridge commands use the subcommand pattern:

```tcl
command subcmd ?args...?
```

### Complete bridge command list

| Command | Subcommands | Purpose |
|---------|-------------|---------|
| `buffer` | save, save-all, list, modified, close, reload, new | Buffer operations |
| `editor` | quit, goto, insert, delete, selection, cursor, undo, redo, find, replace, comment-toggle, diff, git-log, open-search, help, save-session, load-session, cycle-mode | Editor operations |
| `window` | toggle-tree, toggle-left, cycle-focus, focus, split, close, resize, refresh-tree, redraw | Window/panel management |
| `bind` | *(direct)* | Register keybinding |
| `theme` | load, list, get, set | Theme management |
| `terminal` | new-shell, new-kiro, close-tab, prev-tab, next-tab, send, list | Terminal/tab management |
| `kiro` | send, status, cancel | Kiro AI integration |
| `hook` | add, remove, list | Lifecycle hooks |

## Keybinding system

### The `bind` command

```tcl
bind keyspec script
```

`keyspec` is a string describing the key combination. `script` is a
rusticle script body executed when the key is pressed.

### Key spec format

```
modifier+modifier+key
```

Modifiers: `ctrl`, `alt`, `shift`. Key: any printable character, or a
named key (`up`, `down`, `left`, `right`, `home`, `end`, `pageup`,
`pagedown`, `tab`, `enter`, `escape`, `backspace`, `delete`,
`f1`–`f12`, `space`).

Multi-stroke sequences use space separation:

```tcl
bind "ctrl+x ctrl+s" { buffer save }
bind "ctrl+x n"      { terminal new-kiro }
```

### Key spec parsing

```rust
/// A parsed key combination.
#[derive(Clone, Debug, PartialEq, Eq, Hash)]
pub struct KeySpec {
    pub strokes: Vec<Stroke>,
}

/// A single keystroke.
#[derive(Clone, Debug, PartialEq, Eq, Hash)]
pub struct Stroke {
    pub ctrl: bool,
    pub alt: bool,
    pub shift: bool,
    pub key: KeyName,
}

/// Named keys.
#[derive(Clone, Debug, PartialEq, Eq, Hash)]
pub enum KeyName {
    Char(char),
    F(u8),
    Up, Down, Left, Right,
    Home, End, PageUp, PageDown,
    Tab, Enter, Escape, Backspace, Delete, Space,
}
```

### Binding storage and lookup

Bindings are stored in a `BindingTable` that supports both single-stroke
and multi-stroke (chord) lookups.

```rust
/// Stores keybindings and resolves key events to scripts.
pub struct BindingTable {
    /// Single-stroke bindings.
    singles: HashMap<Stroke, BoundAction>,
    /// Multi-stroke prefix table. First stroke → second stroke → action.
    chords: HashMap<Stroke, HashMap<Stroke, BoundAction>>,
}

/// What a keybinding resolves to.
pub struct BoundAction {
    /// The rusticle script to execute.
    pub script: String,
    /// Where this binding was defined (for diagnostics).
    pub source: BindingSource,
}

/// Where a binding came from.
#[derive(Clone, Debug, PartialEq, Eq)]
pub enum BindingSource {
    Default,
    GlobalConfig,
    ProjectConfig,
    Runtime,
}
```

### Key event dispatch

When a crossterm `KeyEvent` arrives:

1. Convert `KeyEvent` → `Stroke`.
2. If a chord prefix is active, look up `chords[prefix][stroke]`.
3. Otherwise, check `chords[stroke]` — if it exists, enter chord-wait
   mode (show "ctrl+x ..." in status bar, wait for next key).
4. Otherwise, look up `singles[stroke]`.
5. If found, evaluate the bound script via `interp.eval(&action.script)`.
6. If not found, pass the key to the focused panel's input handler.

### Rebinding and unbinding

Later `bind` calls for the same keyspec replace the previous binding.
To unbind, bind to an empty script:

```tcl
bind "ctrl+q" {}   ;# unbind ctrl+q
```

### The `theme` command

```tcl
theme load "gruvbox-dark"    ;# loads themes/gruvbox-dark.tcl
theme list                   ;# returns available theme names
theme get "bg"               ;# returns current theme::bg value
theme set "bg" "#1d2021"     ;# override a single property
```

### Theme discovery

`theme load name` searches for theme files in order:

1. `~/.config/kairn/themes/{name}.tcl`
2. Embedded themes compiled via `include_str!`

Embedded themes: `gruvbox-dark`, `gruvbox-light`, `catppuccin`,
`solarized-dark`, `solarized-light`, `one-dark`.

### Theme application

After a theme is loaded, the `ThemeBridge` reads all `theme::*`
context variables and converts them to `txv::Style` values used by
the rendering layer. This happens once at config load and again
whenever `theme load` or `theme set` is called at runtime.

## Concrete Rust types

### ConfigLoader

Orchestrates the full config loading sequence.

```rust
use std::path::{Path, PathBuf};
use rusticle::interpreter::Interpreter;
use rusticle::error::TclError;

/// Embedded default config script.
const DEFAULTS_TCL: &str = include_str!("defaults.tcl");

/// Embedded init template.
const INIT_TEMPLATE: &str = include_str!("init_template.tcl");

/// Result of config loading: the interpreter with all state,
/// plus extracted typed config values.
pub struct ConfigResult {
    pub interp: Interpreter,
    pub config: KairnConfig,
    pub bindings: BindingTable,
    pub theme: ThemeValues,
    pub warnings: Vec<String>,
}

/// Typed config values extracted from the kairn context.
pub struct KairnConfig {
    pub keymap: Keymap,
    pub tab_width: u16,
    pub line_numbers: bool,
    pub auto_save: bool,
    pub theme_name: String,
    pub shell: String,
    pub kiro_command: String,
}

/// Supported keymap modes.
#[derive(Clone, Debug, PartialEq, Eq)]
pub enum Keymap {
    Vi,
    Emacs,
    Classic,
}

/// Loads config files in order and extracts typed values.
pub struct ConfigLoader {
    workspace: PathBuf,
    explicit_config: Option<PathBuf>,
}

impl ConfigLoader {
    pub fn new(workspace: &Path) -> Self;
    pub fn with_explicit_config(self, path: PathBuf) -> Self;
    pub fn load(self) -> Result<ConfigResult, ConfigError>;

    /// Write the init template to ~/.kairnrc.tcl.
    /// Returns the path written, or None if file already exists.
    pub fn init_config() -> Result<Option<PathBuf>, ConfigError>;
}

/// Config loading errors.
#[derive(Debug)]
pub enum ConfigError {
    /// Script evaluation failed.
    Script {
        path: PathBuf,
        error: TclError,
    },
    /// Validation failed (type errors in declared variables).
    Validation {
        path: PathBuf,
        diagnostics: Vec<rusticle::validate::Diagnostic>,
    },
    /// IO error reading config file.
    Io {
        path: PathBuf,
        error: std::io::Error,
    },
}
```

### BridgeRegistry

Registers all bridge commands in the interpreter.

```rust
use std::sync::{Arc, Mutex};
use rusticle::interpreter::Interpreter;

/// Shared mutable state accessible to all bridge commands.
pub struct BridgeState {
    pub buffers: BufferManager,
    pub editor: EditorState,
    pub windows: WindowManager,
    pub terminal: TerminalManager,
    pub bindings: BindingTable,
    pub theme: ThemeValues,
    pub hooks: HookRegistry,
    pub kiro: KiroState,
}

/// Thread-safe handle to bridge state.
pub type BridgeHandle = Arc<Mutex<BridgeState>>;

/// Registers all kairn bridge commands in the interpreter.
/// Returns the shared state handle for use by the event loop.
pub struct BridgeRegistry;

impl BridgeRegistry {
    pub fn register(
        interp: &mut Interpreter,
        state: BridgeHandle,
    ) {
        register_buffer(interp, &state);
        register_editor(interp, &state);
        register_window(interp, &state);
        register_bind(interp, &state);
        register_theme(interp, &state);
        register_terminal(interp, &state);
        register_kiro(interp, &state);
        register_hook(interp, &state);
    }
}
```

### Individual bridge command structs

Each `register_*` function follows the same pattern. The closure
captures a clone of the `BridgeHandle` and dispatches on the
subcommand string.

```rust
/// Buffer bridge: buffer save, buffer list, etc.
fn register_buffer(interp: &mut Interpreter, handle: &BridgeHandle) {
    let h = Arc::clone(handle);
    interp.register_fn("buffer", move |_interp, args| {
        let sub = require_arg(args, 0, "buffer")?;
        let mut st = h.lock().map_err(|_| TclError::new("lock poisoned"))?;
        match sub.as_str() {
            "save" => {
                let path = args.get(1).map(|v| v.as_str().into_owned());
                st.buffers.save_current(path.as_deref())
                    .map(|_| TclValue::Str(String::new()))
                    .map_err(|e| TclError::new(e.to_string()))
            }
            "save-all" => {
                st.buffers.save_all()
                    .map(|_| TclValue::Str(String::new()))
                    .map_err(|e| TclError::new(e.to_string()))
            }
            "list" => {
                let names = st.buffers.list_names();
                Ok(TclValue::List(
                    names.into_iter().map(TclValue::Str).collect()
                ))
            }
            "modified" => {
                let bufid = args.get(1).map(|v| v.as_str().into_owned());
                let m = st.buffers.is_modified(bufid.as_deref());
                Ok(TclValue::Bool(m))
            }
            "close" => {
                let bufid = require_arg(args, 1, "buffer close")?;
                st.buffers.close(&bufid)
                    .map(|_| TclValue::Str(String::new()))
                    .map_err(|e| TclError::new(e.to_string()))
            }
            "reload" => {
                let bufid = args.get(1).map(|v| v.as_str().into_owned());
                st.buffers.reload(bufid.as_deref())
                    .map(|_| TclValue::Str(String::new()))
                    .map_err(|e| TclError::new(e.to_string()))
            }
            "new" => {
                let path = args.get(1).map(|v| v.as_str().into_owned());
                let id = st.buffers.create(path.as_deref());
                Ok(TclValue::Str(id))
            }
            _ => Err(TclError::new(format!(
                "buffer: unknown subcommand \"{sub}\""
            ))),
        }
    });
}

/// Bind bridge: bind "keyspec" { script }
fn register_bind(interp: &mut Interpreter, handle: &BridgeHandle) {
    let h = Arc::clone(handle);
    interp.register_fn("bind", move |_interp, args| {
        let keyspec_str = require_arg(args, 0, "bind")?;
        let script = require_arg(args, 1, "bind")?;
        let keyspec = KeySpec::parse(&keyspec_str)
            .map_err(|e| TclError::new(format!("bind: {e}")))?;
        let mut st = h.lock().map_err(|_| TclError::new("lock poisoned"))?;
        st.bindings.bind(keyspec, BoundAction {
            script,
            source: BindingSource::Runtime,
        });
        Ok(TclValue::Str(String::new()))
    });
}

/// Theme bridge: theme load, theme get, etc.
fn register_theme(interp: &mut Interpreter, handle: &BridgeHandle) {
    let h = Arc::clone(handle);
    interp.register_fn("theme", move |interp, args| {
        let sub = require_arg(args, 0, "theme")?;
        let mut st = h.lock().map_err(|_| TclError::new("lock poisoned"))?;
        match sub.as_str() {
            "load" => {
                let name = require_arg(args, 1, "theme load")?;
                let script = st.theme.find_theme_script(&name)
                    .ok_or_else(|| TclError::new(
                        format!("theme: unknown theme \"{name}\"")
                    ))?;
                // Drop lock before eval to avoid deadlock
                let script = script.to_string();
                drop(st);
                interp.eval(&script)?;
                Ok(TclValue::Str(String::new()))
            }
            "list" => {
                let names = st.theme.available_themes();
                Ok(TclValue::List(
                    names.into_iter().map(TclValue::Str).collect()
                ))
            }
            "get" => {
                let prop = require_arg(args, 1, "theme get")?;
                let val = st.theme.get(&prop)
                    .ok_or_else(|| TclError::new(
                        format!("theme: unknown property \"{prop}\"")
                    ))?;
                Ok(TclValue::Str(val.to_string()))
            }
            "set" => {
                let prop = require_arg(args, 1, "theme set")?;
                let val = require_arg(args, 2, "theme set")?;
                st.theme.set(&prop, &val);
                Ok(TclValue::Str(String::new()))
            }
            _ => Err(TclError::new(format!(
                "theme: unknown subcommand \"{sub}\""
            ))),
        }
    });
}
```

Window, terminal, kiro, and hook bridges follow the same pattern.
Each dispatches on the subcommand string and delegates to the
corresponding manager in `BridgeState`.

### Helper: require_arg

Shared utility for all bridge commands (same as rusticle-tk):

```rust
fn require_arg(
    args: &[TclValue],
    index: usize,
    cmd: &str,
) -> Result<String, TclError> {
    args.get(index)
        .map(|v| v.as_str().into_owned())
        .ok_or_else(|| TclError::new(format!("{cmd}: missing argument {index}")))
}
```

### ThemeValues

```rust
use std::collections::HashMap;

/// Resolved theme colors.
```
