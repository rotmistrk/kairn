# v-011.06 ‚Äî Port Instructions

Parent: v-011-kairn-spec.md

This document describes how to port each module from the existing kairn
codebase (master and feature/mini-ide branches) into the new five-layer
architecture (redesign/f4).

## Status of source branches

| Branch | Contains |
|--------|----------|
| master / redesign/f4 | All current modules: diff, search, session, config, panels, highlight, input, keymap, termbuf, editor.rs (launcher), cli.rs |
| feature/mini-ide (local + remote) | Same file set as master ‚Äî no editor/vi.rs, nav/, or content_search/ modules exist |

The editor/vi.rs, ex.rs, undo.rs, save.rs, nav/\*, and content_search/
modules listed in v-003-architecture.md are **new code to be written**,
not ports. They are documented here for completeness with notes on what
existing code informs their design.

## Fetch instructions

No files need to be fetched from feature/mini-ide. The branch contains
the same source files as master ‚Äî the editor/vi.rs, nav/\*, and
content_search/ modules referenced in v-003-architecture.md do not exist
on any branch and must be written from scratch.

The only branch-specific consideration: if feature/mini-ide diverges
from master in the future with new modules, use:

```sh
# View files on the branch without switching
git show feature/mini-ide:src/<path>

# Extract a single file
git show feature/mini-ide:src/<path> > /tmp/<file>

# Cherry-pick specific commits
git cherry-pick <commit-hash>
```
