# 003.05 — MCP Tier 5: Build & Search Integration

## Objective

Allow Kiro to trigger builds, run commands, and search the project via MCP.

## NOTE: Build and grep infrastructure already exists

- `src/build.rs` — async build/test execution
- `src/grep.rs` — async project-wide search using `ignore` crate
- `src/build_parse.rs` — parse build output for errors
- Results are displayed in ResultsView tabs

## New tools

| Tool | Description |
|------|-------------|
| `run_build` | Trigger project build, return stdout/stderr + parsed errors |
| `run_command` | Run arbitrary shell command, return output |
| `get_build_errors` | Get last build's parsed error list |
| `search_project` | Grep project files (like `:grep`), return matches |
| `search_replace` | Search and replace across files (preview + apply) |

## Implementation

- Build: dispatch via existing build infrastructure, capture output
- Search: reuse existing grep infrastructure (`ignore` crate walker)
- `run_command`: spawn in project directory, capture output (not in PTY)
- `search_replace`: preview returns diffs, apply commits changes via editor

## Safety

- `run_command` limited to project directory
- `search_replace` apply goes through editor buffers (undoable)
- Build output capped at 64KB in MCP response

## Tests

- Scenario test: `run_build` returns parsed errors
- Scenario test: `search_project` finds matches
- Scenario test: `get_build_errors` returns last build results
