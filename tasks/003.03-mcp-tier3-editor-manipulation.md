# 003.03 — MCP Tier 3: Editor Manipulation

## Objective

Allow Kiro to edit buffer content, move cursor, and save files via MCP.

## NOTE: Infrastructure already exists

- `get_tab_content` already reads buffer content (snapshot-based)
- `McpCommandQueue` exists for dispatching mutations
- Editor has `apply_edit()` path used by LSP workspace edits
- `CM_SAVE` command exists internally

## New tools

| Tool | Description |
|------|-------------|
| `edit_buffer` | Apply text edits to open buffer (line range replace) |
| `insert_text` | Insert text at cursor or position |
| `set_cursor` | Move cursor to line:col in specified tab |
| `get_buffer` | Get full buffer content (may already be covered by `get_tab_content`) |
| `save_file` | Save the buffer to disk |

## Implementation

- Add `McpAction` variants for each tool
- `edit_buffer` reuses the same `apply_edit()` path as LSP workspace edits
- Edits go through editor's undo system (user can Ctrl-Z to revert)
- Must handle: file not open (error), file modified (warn)
- Edits applied as single undo group per MCP edit

## Tests

- Scenario test: `edit_buffer` modifies content, verify with `get_tab_content`
- Scenario test: `save_file` persists to disk
- Scenario test: undo after MCP edit reverts the change
