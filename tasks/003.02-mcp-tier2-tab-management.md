# 003.02 — MCP Tier 2: Tab/File Management

## Objective

Allow Kiro to open, create, and close files/tabs via MCP.

## NOTE: Command channel infrastructure already exists

`McpCommandQueue` in `src/mcp/commands.rs` already implements the
mpsc + waker + reply pattern. Currently only routes todo mutations.
Internal commands `CM_OPEN_FILE`, `CM_CLOSE`, `CM_SAVE` exist in the app.
Just need new `McpAction` variants + tool definitions to expose them.

## New tools

| Tool | Description | Internal command |
|------|-------------|-----------------|
| `open_file` | Open existing file in editor | `CM_OPEN_FILE` |
| `create_file` | Create file on disk + open | write + `CM_OPEN_FILE` |
| `close_tab` | Close editor tab by path | `CM_CLOSE` |
| `switch_mode` | Toggle diff / structural | existing mode commands |

## Implementation

- Add `McpAction` variants: `OpenFile`, `CreateFile`, `CloseTab`, `SwitchMode`
- Add tool definitions + handlers (split to `tools_write.rs` if needed for 240-line limit)
- Main thread handler processes new action variants via existing command dispatch

## Tests

- Scenario test: `open_file` via MCP → tab appears in `list_tabs`
- Scenario test: `create_file` creates on disk + opens tab
- Scenario test: `close_tab` removes tab from list
